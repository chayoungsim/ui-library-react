이 페이지는 퍼블리싱 가이드의 실행판입니다. 각 컴포넌트의 실제 동작과 그대로 복사해 쓸 수 있는 마크업을 함께 제공합니다. 모든 예시는 KWCAG 2.2 기준을 통과하도록 작성되어 있으며, 이 페이지 자체도 같은 규칙을 따릅니다.

기준KWCAG 2.2
적용 범위정적 HTML / React / Next.js
연관 문서퍼블리싱 가이드 · 접근성 체크리스트
01

디자인 토큰

색상·간격·radius·z-index는 반드시 토큰을 경유합니다. 컴포넌트에서는 semantic 토큰만 참조하고 --blue-500 같은 primitive를 직접 쓰지 않습니다. 스와치를 클릭하면 변수명이 복사됩니다.

색상 semantic

명도 대비 확인. --color-text-mute(#8B909A)는 흰 배경 기준 약 3.1:1로, 본문 텍스트에는 사용할 수 없습니다. 18pt 이상 큰 글씨나 보조 라벨에만 쓰고, 읽어야 하는 텍스트는 --color-text 이상을 사용하세요. (KWCAG 5.4.3)

02

타이포그래피

본문은 Pretendard, 코드·수치·라벨은 JetBrains Mono를 씁니다. 글자 크기는 rem으로 지정하고 html { font-size: 62.5% } 트릭은 사용하지 않습니다.

네이밍 규칙

BEM 축약형을 기본으로 하고, 접두사로 역할을 구분합니다.

클래스 접두사
접두사 용도 예시
ui-공통 UI 라이브러리 컴포넌트.ui-button
(없음)프로젝트 전용 컴포넌트.card

권장

<article class="card">
  <h3 class="card__title">제목</h3>
  <p class="card__text">설명</p>
  <button class="ui-button js-card-open">열기</button>
</article>

역할로 이름을 붙이고, 스타일 클래스와 JS 훅을 분리합니다.

금지

<div class="box1">
  <div class="red-text font-14 mt-20">제목</div>
  <div class="txt">설명</div>
  <div class="btn-primary" onclick="open()">열기</div>
</div>

red-text는 색이 바뀌는 순간 거짓말이 됩니다. 스타일 클래스를 JS 훅으로 쓰면 디자인 변경 시 기능이 깨집니다.

회사 메일 주소를 입력하세요

약관 동의 
<div class="ui-field has-error">
  <label class="ui-label" for="phone">
    연락처
    <span class="req" aria-hidden="true">*</span>
    <span class="sr-only">(필수)</span>
  </label>

  <input
    class="ui-input"
    id="phone"
    type="tel"
    autocomplete="tel"
    aria-describedby="phone-help phone-error"
    aria-invalid="true">

  <p class="ui-help" id="phone-help">하이픈 없이 입력하세요</p>

  • label의 for와 input의 id를 반드시 연결합니다.
  • 필수 표시를 빨간 별표로만 하면 색을 구분하지 못하는 사용자가 인식할 수 없습니다. sr-only로 "(필수)" 텍스트를 함께 넣습니다.

  • role="alert"을 붙이면 스크린리더가 즉시 읽어줍니다.

탭

탭 목록은 Tab 키로 하나의 단위처럼 취급하고, 탭 사이 이동은 좌우 방향키로 합니다. 아래 탭에 초점을 두고 ← →를 눌러보세요.

C-03 ui-tabs 6.1.1 6.1.2 roving tabindex
태그는 역할로 선택합니다. 클릭 시 이동하면 a, 동작을 실행하면 button입니다.
색상·간격·z-index는 토큰을 경유합니다. 중첩은 최대 3단계까지만 허용합니다.

아코디언

트리거는 반드시 button입니다. 닫힌 패널은 hidden으로 처리해 초점이 보이지 않는 요소로 들어가지 않게 합니다.

C-04 ui-accordion 6.1.1 aria-expanded

규칙을 바꾸려면 문서를 먼저 고치고 팀에 공유합니다. 코드가 먼저 바뀌고 문서가 따라오면 이 문서는 죽습니다.

아니요. 신규 프로젝트부터 적용하고, 기존 프로젝트는 손대는 파일에 한해 점진적으로 맞춥니다.

판단해서 작업한 뒤, 같은 판단을 두 번 하게 되면 그때 문서에 추가합니다. 
<div class="ui-accordion" data-accordion>
  <div class="ui-accordion__item">
    <h3>
      <button type="button" class="ui-accordion__trigger"
              aria-expanded="false" aria-controls="acc-1">
        질문 제목
        <span class="ui-accordion__icon" aria-hidden="true"></span>
      </button>
    </h3>
    <div class="ui-accordion__panel" id="acc-1" hidden>
      답변 내용
    </div>
  </div>
</div>
09

모달

가장 까다로운 컴포넌트입니다. 열기·닫기만 되면 완성이 아니라, 초점 관리까지 되어야 완성입니다.

C-05 ui-modal 6.1.2 초점 복귀 7.2.1 닫기 focus trap
열고 나서 Tab과 Esc를 눌러보세요 
<div class="ui-modal-backdrop" data-modal hidden> 
  <div class="ui-modal" id="demo-modal"
      role="dialog" aria-modal="true"
      aria-labelledby="demo-modal-title" >
    <div class="ui-modal__head">
      <h2 class="ui-modal__title" id="demo-modal-title">변경 사항 저장</h2>
      <button type="button" class="ui-modal__close" data-modal-close>
        <span aria-hidden="true">×</span>
        <span class="sr-only">닫기</span>
      </button>
    </div>
    <div class="ui-modal__body">...</div>
    <div class="ui-modal__foot">
      <button type="button" class="ui-button ui-button--secondary" data-modal-close>취소</button>
      <button type="button" class="ui-button ui-button--primary">저장하기</button>
    </div>
  </div>
</div>
  • 열 때 초점을 모달 내부로 옮기고, 닫을 때 원래 트리거 버튼으로 되돌립니다.
  • Tab이 모달 밖으로 나가지 않도록 가둡니다(focus trap).
  • Esc로 닫힙니다.
  • 배경 스크롤을 잠급니다.
  • 배경 콘텐츠에 inert 또는 aria-hidden="true"를 적용합니다.
React라면 직접 구현하지 마세요. Radix UI나 React Aria 같은 헤드리스 라이브러리를 쓰고 스타일만 입히는 편이 낫습니다. 포커스 트랩은 직접 만들면 엣지 케이스가 끝없이 나옵니다.

알림

색만으로 상태를 구분하지 않습니다. 기호나 아이콘, 텍스트를 함께 제공합니다.

C-06 ui-alert 5.4.1 색 무관 role=status
i 변경 사항은 30초마다 자동 저장됩니다.
✓ 저장했습니다.
! 연결이 끊겼습니다. 네트워크를 확인한 뒤 다시 시도하세요. 
<!-- 일반 상태 알림 -->
<div class="ui-alert ui-alert--success" role="status">
  <span class="ui-alert__mark" aria-hidden="true">✓</span>
  <span>저장했습니다.</span>
</div>

<!-- 즉시 알려야 하는 오류 -->
<div class="ui-alert ui-alert--error" role="alert">
  <span class="ui-alert__mark" aria-hidden="true">!</span>
  <span>연결이 끊겼습니다. 네트워크를 확인한 뒤 다시 시도하세요.</span>
</div>
오류 메시지는 사과하지 않고, 모호하지 않게. "죄송합니다. 문제가 발생했습니다"가 아니라 무엇이 잘못됐고 어떻게 해결하는지를 씁니다.

유틸리티

모든 프로젝트에 들어가는 최소한의 공통 코드입니다.

U-01 sr-only · skip-nav · reduced-motion 6.4.1 필수 포함 
/* 시각적으로 숨기되 스크린리더는 읽음 */
.sr-only {
  position: absolute;
  width: 1px; height: 1px;
  padding: 0; margin: -1px;
  overflow: hidden;
  clip-path: inset(50%);
  white-space: nowrap;
}

/* 건너뛰기 링크 — 초점을 받으면 반드시 보여야 함 */
#accessibility {
  position: relative;
  margin: 0;
  padding: 0;
}

#accessibility {
  a {
    display: block;
    height: 1px;
    width: 1px;
    margin-bottom: -1px;
    overflow: hidden;
    text-align: center;
    color: #000;
    white-space: nowrap;
    &:focus, &:active {
      position: absolute;
      height: auto;
      padding: 5px;
      margin-bottom: 10px;
      width: 100%;
      background: #fff;
      z-index: 9999;
    }
  }
}

/* 초점 표시 */
:focus-visible {
  outline: var(--focus-ring);
  outline-offset: var(--focus-offset);
}

/* 모션 축소 설정 존중 */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
display: none은 스크린리더도 읽지 못합니다. 시각적으로만 숨기고 싶을 때는 반드시 sr-only를 사용하세요.
