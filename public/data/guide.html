컴포넌트 마크업 레퍼런스

이 페이지의 마크업을 복사해 그대로 사용합니다. 네이밍·접근성 규칙 등 전체 컨벤션은 docs/MARKUP-GUIDE.md를 참고하세요.

색상 토큰 tokens / color

색상은 항상 SCSS 변수로 참조합니다.

TOKEN / COLOR
--color-primary#2B50D8
--color-link#1C2530
--color-text#66707C
--color-border#E2E6EB
--color-error#D64545
color: var(--color-link);
background: var(--color-background);
border-color: var(--color-border);

타이포그래피 base / type

h1은 페이지당 1개, heading 레벨은 건너뛰지 않습니다.

BASE / TYPE

H2 페이지 타이틀 32px

H3 섹션 타이틀 24px

H4 소제목 18px

본문 15px — 프로젝트의 기본 본문 크기입니다. 행간은 1.6을 유지하고, 한 줄이 60자를 크게 넘지 않게 컨테이너 폭을 제한합니다.

캡션 13px — 보조 설명, 날짜 등에 사용합니다. 
<h2>페이지 타이틀</h2>
<h3>섹션 타이틀</h3>
<p>본문 텍스트</p>
<small>캡션 텍스트</small>

버튼 components / button

클릭 동작은 반드시 button, 페이지 이동은 a를 사용합니다. 변형은 --modifier로 확장합니다.

BTN / VARIANTS 
<button type="button" class="btn btn--primary">기본 버튼</button>
<button type="button" class="btn btn--secondary">보조 버튼</button>
<button type="button" class="btn btn--outline">삭제</button>
<button type="button" class="btn btn--sm">작은 버튼</button>
<button type="button" class="btn" disabled>비활성</button>

type="button"을 항상 명시합니다. 폼 안에서 기본값이 submit이라 의도치 않은 전송이 발생할 수 있습니다.

폼 components / form

모든 입력 요소는 label과 for/id로 연결하고, 오류 메시지는 aria-describedby로 연결합니다.

FORM / FIELD

이메일 형식이 올바르지 않습니다.

수신 동의 
<div class="form-field has-error">
  <label class="form-field__label" for="user-email">
    이메일 <em class="form-field__required" aria-hidden="true">*</em>
  </label>
  <input class="form-field__input" id="user-email" type="email"
         required aria-describedby="user-email-error" aria-invalid="true">
  <p class="form-field__message" id="user-email-error">이메일 형식이 올바르지 않습니다.</p>
</div>

테이블 components / table

데이터 표에만 사용합니다. caption과 scope를 반드시 포함합니다.

TABLE / DATA
2026년 상반기 프로젝트 현황
프로젝트담당상태
브랜드 사이트 리뉴얼퍼블파트진행 중
이벤트 프로모션퍼블파트완료
어드민 개편FE파트대기
 
<table class="tbl">
  <caption>2026년 상반기 프로젝트 현황</caption>
  <thead>
    <tr><th scope="col">프로젝트</th><th scope="col">담당</th><th scope="col">상태</th></tr>
  </thead>
  <tbody>
    <tr><td>브랜드 사이트 리뉴얼</td><td>퍼블파트</td><td>진행 중</td></tr>
  </tbody>
</table>

카드 components / card

카드 목록은 ul > li 구조를 유지합니다. 카드 전체 클릭이 필요하면 제목의 a를 확장하는 방식을 사용합니다.

CARD / LIST

  • 카드 타이틀

    두 줄 내외의 짧은 설명이 들어갑니다.

  • 카드 타이틀

    두 줄 내외의 짧은 설명이 들어갑니다.

<ul class="card-list">
  <li class="card">
    <img class="card__thumb" src="./img/thumb.jpg" alt="" width="320" height="180" loading="lazy">
    <div class="card__body">
      <h3 class="card__title"><a href="/detail">카드 타이틀</a></h3>
      <p class="card__text">짧은 설명</p>
    </div>
  </li>
</ul>

탭 components / tabs

WAI-ARIA 탭 패턴을 따릅니다. 좌우 방향키로 탭 간 이동이 가능해야 합니다.

TABS / ARIA
공지사항 패널 콘텐츠입니다.
이벤트 패널 콘텐츠입니다.
FAQ 패널 콘텐츠입니다.
<div class="tabs js-tabs">
  <div class="tabs__list" role="tablist" aria-label="예시 탭">
    <button class="tabs__tab" role="tab" id="tab-1" aria-controls="panel-1"
            aria-selected="true" type="button">공지사항</button>
    <button class="tabs__tab" role="tab" id="tab-2" aria-controls="panel-2"
            aria-selected="false" tabindex="-1" type="button">이벤트</button>
  </div>
  <div class="tabs__panel" role="tabpanel" id="panel-1" aria-labelledby="tab-1" tabindex="0">...</div>
  <div class="tabs__panel" role="tabpanel" id="panel-2" aria-labelledby="tab-2" tabindex="0" hidden>...</div>
</div>

아코디언 components / accordion

네이티브 details/summary를 사용합니다. JS 없이 동작하고 키보드 접근성이 기본 제공됩니다.

ACC / NATIVE
배송은 얼마나 걸리나요?
평일 오후 2시 이전 주문 시 당일 출고되며, 평균 1~2일 소요됩니다.
교환/환불 규정이 궁금해요
수령 후 7일 이내 신청 가능합니다.
<details class="acc">
  <summary>배송은 얼마나 걸리나요?</summary>
  <div class="acc__body">평일 오후 2시 이전 주문 시 당일 출고됩니다.</div>
</details>

모달 components / modal

네이티브 dialog + showModal()을 사용합니다. 포커스 트랩과 ESC 닫기가 기본 제공됩니다.

MODAL / DIALOG

변경사항을 저장할까요?

저장하지 않으면 작성 중인 내용이 사라집니다.

 
<button type="button" class="btn js-modal-open" data-modal="demo-modal">모달 열기</button>

<dialog class="modal" id="demo-modal" aria-labelledby="demo-modal-title">
  <div class="modal__inner">
    <h3 class="modal__title" id="demo-modal-title">변경사항을 저장할까요?</h3>
    <p class="modal__text">저장하지 않으면 작성 중인 내용이 사라집니다.</p>
    <div class="modal__actions">
      <button type="button" class="btn btn--secondary js-modal-close">취소</button>
      <button type="button" class="btn js-modal-close">저장</button>
    </div>
  </div>
</dialog>

<script>
  document.querySelector('.js-modal-open').addEventListener('click', (e) => {
    document.getElementById(e.currentTarget.dataset.modal).showModal();
  });
</script>

접근성 패턴 base / a11y

숨김 텍스트, 스킵 링크 등 프로젝트 공통 접근성 유틸입니다. 이 페이지 최상단에서 Tab 키를 누르면 스킵 링크가 나타납니다.

A11Y / UTILS
← 아이콘 버튼: 화면에는 아이콘만, 스크린 리더에는 “검색”으로 읽힙니다. 
<!-- 스킵 링크 : body 최상단 -->
<p id="accessibility"><a href="#content">본문 바로가기</a></p>

<!-- 아이콘 전용 버튼 -->
<button type="button" class="btn">
  <span aria-hidden="true">🔍</span>
  <span class="sr-only">검색</span>
</button>

<!-- 숨김 텍스트 유틸 -->
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  margin: -1px;
  padding: 0;
  border: 0;
  clip-path: inset(50%);
  overflow: hidden;
  white-space: nowrap;
}
